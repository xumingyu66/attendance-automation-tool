  考勤自动化桌面应用 — 需求分析

  一、业务背景

  将 考勤报表.xls（原始打卡数据）自动填充到 文件考勤.xlsx（考勤汇总模板），并生成格式化的 Word 文档作为交付物。

  二、核心业务规则

  ┌──────────┬────────────────────────────────────────────────────────────────────┐
  │   规则   │                                说明                                │
  ├──────────┼────────────────────────────────────────────────────────────────────┤
  │ 工时计算 │ 只要当天有打卡痕迹，无论时长，计 2 小时                            │
  ├──────────┼────────────────────────────────────────────────────────────────────┤
  │ 数据来源 │ 考勤报表.xls 的前 2 个子表跳过，从 "1"、"2"、"3"… 子表读取         │
  ├──────────┼────────────────────────────────────────────────────────────────────┤
  │ 人员匹配 │如果考勤报表中的人员在文件考勤表中找不到则跳过
  ├──────────┼────────────────────────────────────────────────────────────────────┤
  ├──────────┼────────────────────────────────────────────────────────────────────┤
  │ 汇总列   │ 每行最后一列为当月总工时，无数据填 0                               │
  └──────────┴────────────────────────────────────────────────────────────────────┘

  三、输入输出

  ┌──────────────────┐     ┌──────────────────┐
  │  考勤报表.xls     │     │  文件考勤.xlsx    │
  │  (原始打卡数据)   │     │  (考勤汇总模板)   │
  └────────┬─────────┘     └────────┬─────────┘
           │                        │
           └──────────┬─────────────┘
                      ▼
           ┌──────────────────┐
           │   桌面应用 (GUI)   │
           │  PySide6 + qasync │
           └────────┬─────────┘
                    │
          ┌─────────┴─────────┐
          ▼                   ▼
  ┌───────────────┐   ┌───────────────┐
  │ 文件考勤.xlsx  │   │ 考勤报告.docx  │
  │ (填好的)       │   │ (Word 文档)   │
  └───────────────┘   └───────────────┘

  四、功能模块

  模块 1：文件选择

  - 分别选择 考勤报表.xls 和 文件考勤.xlsx
  - 支持拖拽或文件对话框
  - 显示所选文件路径

  模块 2：考勤数据解析（Service 层）

  - 读取 .xls 文件（使用 xlrd）
  - 跳过前 2 个子表
  - 遍历子表 "1"、"2"、"3"…，提取每日打卡记录，只用统计改日用户是否有打卡记录即可
  - 输出数据结构：{员工名: {日期: bool}}

  模块 3：考勤填充（Service 层）

  - 读取 文件考勤.xlsx（使用 openpyxl）
  - 定位每个员工的日期单元格区域
  - 根据考勤报表数据填入 2 或留空
  - 重新计算每月汇总列
  - 保存为新文件

  模块 4：Word 文档生成（Service 层）

  - 使用 python-docx
  - 按参考图片格式生成考勤报告
  - 包含统计汇总信息

  模块 5：GUI 界面（UI 层）

  - 主窗口：文件选择区 + 配置区 + 一键执行按钮
  - 进度条显示处理状态
  - 结果预览/日志区域
  - 菜单栏 + 状态栏

  模块 6：设置管理

  - QSettings 记住上次选择的文件路径
  - 可配置默认输出目录

  五、技术架构（遵循 building-qt-apps 规范）

  ┌─────────────────────────────────────┐
  │  UI Layer                            │
  │  MainWindow, FileDialogs, ProgressBar│
  ├─────────────────────────────────────┤
  │  Manager Layer                       │
  │  AttendanceManager                   │
  │  - 协调解析→填充→生成流程            │
  │  - 发射进度/完成/错误信号             │
  ├─────────────────────────────────────┤
  │  Service Layer                       │
  │  AttendanceParser (解析考勤报表)      │
  │  AttendanceFiller (填充考勤文件)      │
  │  WordReportGenerator (生成Word报告)   │
  ├─────────────────────────────────────┤
  │  Wrapper Layer                       │
  │  XlsReaderWrapper (封装xlrd)         │
  │  XlsxWrapper (封装openpyxl)          │
  │  DocxWrapper (封装python-docx)       │
  └─────────────────────────────────────┘

  - GUI 框架：PySide6
  - 异步支持：qasync，耗时操作用 ThreadPoolExecutor 避免阻塞 UI
  - Excel .xls 读取：xlrd（Wrapper 封装）
  - Excel .xlsx 读写：openpyxl（Wrapper 封装）
  - Word 生成：python-docx（Wrapper 封装）
  - 日期处理：calendar + datetime 标准库

  六、数据流

  1. 用户选择两个输入文件
  2. 点击"开始处理"
  3. [ThreadPoolExecutor] 解析考勤报表 → 得到 {姓名: {日期: 有打卡}}
  4. [ThreadPoolExecutor] 读取考勤模板 → 定位每人每日期单元格
  5. [ThreadPoolExecutor] 填充工时 + 更新日期行 + 计算汇总
  6. [ThreadPoolExecutor] 生成 Word 报告
  7. 保存两个输出文件
  8. UI 显示完成，可打开输出目录
